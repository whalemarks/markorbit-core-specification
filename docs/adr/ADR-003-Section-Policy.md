# ADR-003 — Section Policy

Version: 1.0

Status: Accepted

Date: 2026-07

---

# Context

Specification documents have different responsibilities.

Not every specification requires identical sections.

This ADR defines section requirements.

---

# Decision

Sections are divided into three categories.

---

# Required Sections

Every Specification shall include:

- Title
- Version
- Status
- Purpose
- Summary

Metadata shall be complete.

---

# Recommended Sections

Include when appropriate:

- Depends
- Related
- Scope
- Responsibilities
- Architecture
- Validation
- Characteristics

---

# Optional Sections

Use only when meaningful.

Examples:

- Runtime
- Examples
- Compatibility
- Extension
- Relationships
- Categories
- Non Goals

Optional sections SHALL NOT be added merely for consistency.

---

# Editorial Rule

Editorial review shall validate Required sections.

Missing Recommended sections may be suggested.

Missing Optional sections are not defects.

---

# Implementation

Editorial tools shall distinguish between:

Required

Recommended

Optional

Repository consistency shall respect document responsibility.

---

# Consequences

Specifications remain concise while maintaining a consistent publishing standard.

Consistency supports architecture rather than replacing it.
