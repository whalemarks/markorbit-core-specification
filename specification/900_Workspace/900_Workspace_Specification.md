# Purpose

Professional systems do not operate in isolation.

Every fact, judgment, execution and interaction exists within an organizational context.

The purpose of Workspace is to define the organizational context in which the MarkOrbit Operating System operates.

Workspace answers one question.

> **Who is performing professional work, under which organizational rules?**

---

# Workspace Philosophy

Workspace does not execute work.

Workspace does not coordinate work.

Workspace provides organizational context.

Everything else operates inside Workspace.

---

# Scope

Workspace defines:

- organizations
- teams
- permissions
- branding
- localization
- policies
- integrations
- governance
- configuration

Workspace is organization-specific.

Professional architecture remains universal.

---

# Responsibilities

Workspace owns:

- organizational identity
- organizational configuration
- organizational policies
- organizational permissions
- organizational integrations

Workspace does not own:

- knowledge
- reasoning
- execution
- coordination
- interaction

---

# Summary

Workspace defines the organizational context of the MarkOrbit Professional Operating System.

It encapsulates the complete professional architecture—Persistence, Brain, Intelligence, Capability, Workflow and Guide—within an organization-specific context of policies, permissions, integrations and governance.

Workspace enables organizations to operate independently while sharing one universal professional architecture.
