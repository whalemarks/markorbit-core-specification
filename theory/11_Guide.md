# Purpose

Professional capability alone is not sufficient.

Users need a natural way to access professional capability.

The purpose of Guide is to understand user intent, manage interaction context, and connect users with the appropriate professional capabilities.

Guide answers one question.

> **What does the user actually need?**

---

# Guide Philosophy

Guide does not perform professional work.

Guide enables professional work.

Guide understands.

Guide organizes.

Guide recommends.

Guide coordinates.

Workflow executes.

Capability performs.

---

# The Interaction Pipeline

Guide continuously transforms interaction into execution.

```
User

↓

Intent

↓

Context

↓

Guide

↓

Capability Selection

↓

Workflow

↓

Execution
```

Guide bridges people and professional systems.

---

# What Is Guide

Guide is the interaction layer of MarkOrbit.

Its responsibilities are to:

- understand intent
- collect information
- manage context
- recommend capabilities
- select workflows
- explain professional knowledge

Guide never owns professional capability.

Guide orchestrates access to professional capability.

---

# Core Responsibilities

Guide owns exactly one responsibility:

**Interaction and Context Management**

Guide does not own:

- knowledge
- judgment
- execution
- workflow state

These belong to other Core components.

---

# Intent Understanding

Every interaction begins with user intent.

Examples include:

- Register a trademark
- Renew a trademark
- Buy a trademark
- Analyze a portfolio
- Ask a legal question

Intent may be:

- explicit
- incomplete
- ambiguous
- evolving

Guide continuously refines intent.

---

# Context Management

Guide manages context throughout the interaction.

Context may include:

- user information
- workspace configuration
- previous conversations
- current entities
- business objectives
- selected capabilities

Context belongs to Guide.

Execution state belongs to Workflow.

---

# Progressive Understanding

Guide should never assume complete understanding.

Instead it progressively improves understanding.

```
Intent

↓

Question

↓

Answer

↓

Updated Context

↓

Next Question

↓

Execution Ready
```

Guide asks only when necessary.

Every question should reduce uncertainty.

---

# Capability Selection

Guide does not perform professional work.

Guide selects the most appropriate Capabilities.

Selection considers:

- user intent
- current context
- workspace policy
- available capabilities
- intelligence recommendations

Capabilities remain independent.

Guide coordinates them.

---

# Workflow Selection

Once sufficient information has been collected,

Guide selects the appropriate Workflow.

Execution begins only when:

- intent is clear
- required information is complete
- execution conditions are satisfied

Guide prepares.

Workflow executes.

---

# Knowledge Assistance

Guide explains professional concepts through Brain.

Guide may:

- explain terminology
- explain legal requirements
- explain recommendations
- explain risks

Guide never creates knowledge.

Guide retrieves and presents knowledge.

---

# Intelligence Assistance

Guide consumes Intelligence Products.

Examples include:

- recommended countries
- recommended classes
- filing strategy
- risk alerts
- commercial opportunities

Guide explains recommendations.

Guide does not generate judgment.

---

# Human Collaboration

Guide supports collaboration between humans and AI.

Guide may:

- prepare information
- summarize conversations
- request confirmation
- transfer to professionals

Professional responsibility always remains clear.

---

# Memory

Guide maintains interaction memory.

Memory supports:

- conversation continuity
- preference preservation
- context reconstruction

Memory belongs to the interaction process.

It is not part of Entity history.

---

# Personalization

Guide itself has no personality.

Instead,

Guide adapts according to:

- workspace configuration
- user preferences
- product design
- interaction policies

Interaction style may vary.

Professional principles remain consistent.

---

# Guide Independence

Guide is independent of:

- LLM providers
- AI frameworks
- user interfaces
- communication channels

Guide may operate through:

- web
- mobile
- API
- voice
- messaging platforms

Interaction channels evolve.

Guide remains stable.

---

# Guide and Brain

Brain provides knowledge.

Guide explains knowledge.

Guide never modifies Brain.

---

# Guide and Intelligence

Intelligence provides recommendations.

Guide communicates recommendations.

Guide never replaces professional judgment.

---

# Guide and Capability

Capability performs professional work.

Guide selects Capabilities.

Guide never implements Capabilities.

---

# Guide and Workflow

Guide prepares execution.

Workflow executes business.

Guide may continue asking questions until execution becomes possible.

Workflow begins only after Guide completes preparation.

---

# Guide and Workspace

Workspace configures Guide.

Examples include:

- supported languages
- service policies
- regional rules
- permission models
- available capabilities

Guide behaves consistently across Workspaces while respecting local configuration.

---

# Design Principles

Intent before execution.

Context before capability.

Guide before Workflow.

Knowledge before recommendation.

Progressive understanding before automation.

Professional consistency before conversational creativity.

---

# Summary

Guide is the interaction system of MarkOrbit.

It understands user intent, manages interaction context, explains professional knowledge, selects capabilities and prepares business execution.

Guide never replaces professional capability.

It enables users to access professional capability naturally and consistently across the entire Trademark Universe.
