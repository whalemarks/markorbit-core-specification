# What Is a Capability

A Capability is an independently executable professional ability.

A Capability performs exactly one professional responsibility.

Examples include:

- Recommend Country
- Recommend Class
- Recommend Goods
- Generate Filing Package
- Validate Applicant
- Check Deadline
- Calculate Official Fees
- Generate Office Action Draft

One Capability.

One Responsibility.

---

# Capability Characteristics

Every Capability should satisfy the following principles.

## Atomic

A Capability performs one professional task.

It should not contain business workflows.

---

## Reusable

The same Capability should serve:

- Guide
- Workflow
- API
- Workspace
- Products

Professional capability should never be duplicated.

---

## Composable

Capabilities may be combined into larger business processes.

Complex work is achieved through composition rather than larger Capabilities.

---

## Stateless

Capability owns no business state.

Execution state belongs to Workflow.

Business context belongs to Guide.

Operational configuration belongs to Workspace.

Capability focuses only on professional execution.

---

## Versioned

Professional knowledge evolves.

Capability therefore evolves.

Every Capability should have a version.

Older versions remain reproducible.

New versions continuously improve.

---

## Testable

Every Capability must be independently testable.

A Capability that cannot be tested should not enter the Capability Catalog.

Testing includes:

- input validation
- expected output
- edge cases
- regression
- performance

Professional trust depends on repeatable execution.

# Capability Catalog

Capability Catalog is the permanent collection of all professional capabilities.

It is one of the most valuable long-term assets of MarkOrbit.

Every Capability should include:

- unique identity
- version
- description
- inputs
- outputs
- dependencies
- test status
- supported products

The catalog continuously grows.

Capabilities are rarely removed.

---

# Capability Composition

Large business operations are composed from multiple Capabilities.

Example

```
US Trademark Filing

↓

Recommend Country

↓

Recommend Classes

↓

Recommend Goods

↓

Validate Applicant

↓

Calculate Fees

↓

Generate Filing Package
```

Each Capability remains independent.

The Workflow coordinates them.

---

# Capability and Brain

Brain provides knowledge.

Capability operationalizes knowledge.

Every Capability should reference the Brain Objects upon which it depends.

Brain remains the source of professional understanding.

---

# Capability and Intelligence

Intelligence determines what should happen.

Capability determines how to perform it.

Judgment precedes execution.

Execution never replaces judgment.

---

# Capability and Guide

Guide selects Capabilities.

Guide provides:

- user intent
- workspace context
- conversation context

Guide never implements Capability.

Guide orchestrates Capability.

---

# Capability and Workflow

Workflow executes business processes.

Workflow composes Capabilities.

Workflow owns:

- execution order
- execution state
- approvals
- retries
- human intervention

Capability remains independent of business processes.

---

# Capability and Workspace

Workspace configures Capability.

Examples include:

- permissions
- pricing
- regional policies
- organizational rules

Workspace never changes the professional definition of a Capability.

---

# Capability Independence

Capability should remain independent of:

- products
- AI models
- user interfaces
- storage technologies

Professional capability should survive technological evolution.

---

# Feedback Network

Every execution generates feedback.

Feedback may include:

- execution success
- customer acceptance
- attorney review
- business outcome
- quality metrics

Feedback continuously improves:

- Brain
- Intelligence
- Capability

Capability Catalog and Feedback Network together form the long-term competitive advantage of MarkOrbit.

# Design Principles

Knowledge before Capability.

Judgment before Capability.

Capability before Workflow.

Atomic before Complex.

Composition before Duplication.

Testing before Release.

Evolution before Replacement.

# Summary

Capability is the professional execution system of MarkOrbit.

It transforms professional judgment into reusable execution.

Capabilities are atomic, reusable, composable, versioned and testable.

They accumulate within the Capability Catalog and continuously improve through the Feedback Network.

Professional capability becomes permanent digital infrastructure.
